How it works

Draw a causal loop diagram (e.g., in Kumu)→ Annotate the Excel export→ Simulate under uncertainty

Causal loop diagrams capture how experts and stakeholders believe a system works, but they are qualitative. SIP turns them into computational system dynamics models: every causal link gets a sampled coefficient, and the system is simulated many times. That one uncertainty ensemble then feeds three complementary lines of analysis:

1 Decide — find leverage
Rank interventions → Optimize the budget

Which levers move the outcome, and the best portfolio of them under a budget.

2 Explain — the mechanism
Loops That Matter

Which feedback loops drive the behaviour over time — and why an intervention works or stalls.

3 Stress-test — robustness
Sensitivity analysis → Scenario discovery

Which uncertain assumptions your conclusions hinge on, and which combinations of conditions make them fail.

These lines are a web, not a straight pipe — they inform one another. Global sensitivity analysis ranks which uncertain assumptions matter globally; scenario discovery then localises the danger — the combination of conditions under which the outcome goes bad — so it naturally follows sensitivity analysis and shares the same ensemble (the parameters it isolates should echo the top sensitivity drivers, a built-in cross-check). Loops That Matter runs in parallel: it explains the mechanism, and can be pointed inside a discovered failure region to reveal which loop dominance produces it — while the loops it flags suggest which links to probe with sensitivity analysis. And because the robustness track tells you where measurement effort pays off, its findings feed back to sharpen the ranking and optimization. In practice you iterate: decide, explain, stress-test, refine.

What it can do

CLD → dynamic model

Reads a Kumu-style Excel workbook (variables, links, polarities) and builds a stock–auxiliary–constant ODE model automatically.

Simulation under uncertainty

Samples unknown link coefficients and simulates every candidate intervention across hundreds of parameter draws.

Intervention ranking

Effect distributions per intervention, pairwise comparisons with bootstrapped confidence intervals, and sensitivity analysis of the driving links.

Budget optimization

Optimally splits a budget across all interventions at once — expenditure-space formulation with multi-start SLSQP that scales to dozens of interventions and reports near-optimal alternatives.

Loops That Matter

Scores every feedback loop's contribution to the model's behavior over time (Schoenberg–Davidsen–Eberlein method).

Custom equations

Make any relationship nonlinear straight from the Excel file: sampled parameters (#1), intervention placement ($), and numpy functions.

Global sensitivity analysis

Variance-based Sobol indices (S1/ST with BCa confidence intervals) plus moment-independent measures (Borgonovo δ, PAWN) that capture shifts in the whole output distribution.

Scenario discovery

PRIM & CART localise the combinations of conditions that produce a bad outcome, as interpretable boxes/rules with density and coverage.

Quick start

git clone https://github.com/vvvasconcelos/SystemsInsightPipeline.git
cd SystemsInsightPipeline
pip install -e .
from sip_systemsinsightpipeline import Extract, SDM, SDMOptimizer

s = Extract("tutorials/Minimal.xlsx").extract_settings()
s.seed, s.N, s.t_end, s.time_unit = 12345, 100, 10, "years"
s.parameter_value_aux = s.parameter_value_stocks = 0.1

sdm = SDM(s)
sdm.run_simulations()
effects = sdm.get_intervention_effects()

result = SDMOptimizer(sdm).optimize_intervention_intensities(
    params=sdm.sample_model_parameters(),
    costs=[1.0] * len(s.intervention_variables),
    budget=1.0, maximize=True, n_starts=8, seed=1,
)
