Sensitivity & scenario discovery

Two standard uncertainty tools that round out the workflow: which assumptions your conclusion is most sensitive to (global sensitivity analysis), and which combinations of conditions produce a bad outcome (scenario discovery).

This builds directly on the Insulation tutorial and uses the same SEVEN-project peak-energy model. Read that one first for the modelling workflow (diagram → annotate → simulate → rank → optimize → loops). A runnable notebook version of this page is in tutorials/GSA_and_Scenario_Discovery.ipynb.

0Where these fit

Global sensitivity analysis

Ranks the model's uncertain parameters by how much each drives the variance of the outcome. Unlike intervention ranking (which ranks the levers you could pull) and unlike one-factor-at-a-time analysis, it accounts for the full input ranges and their interactions at once.

Scenario discovery

Localises the region of the input space where an outcome of concern concentrates — "under what combination of conditions does it go wrong?". The complement of GSA: GSA ranks drivers globally; scenario discovery draws a box around the danger.

1Global sensitivity analysis (Sobol)

Every causal link in a qualitative diagram has an uncertain strength. GSA asks: of all those uncertain strengths, which ones actually matter for the outcome? It draws a structured (Saltelli) sample over the parameters — the same uniform priors the simulator already uses — runs the model once per design point under a chosen intervention scenario, and decomposes the outcome variance into contributions per parameter.

from sip_systemsinsightpipeline import Extract, SDM
from sip_systemsinsightpipeline.plots import plot_gsa

s = Extract("Insulation_PeakEnergy.xlsx").extract_settings()
s.seed = 0; s.N = 1; s.t_end = 15; s.time_unit = "years"
s.parameter_value_aux = 0.3; s.parameter_value_stocks = 0.1
sdm = SDM(s)
voi = s.variable_of_interest[0]

# total runs = n * (d + 2); keep n modest on a large model for a first look
gsa_df = sdm.run_GSA(voi, n=128, seed=1)
fig = plot_gsa(gsa_df, top=12)

How to read it S1 (first-order) is the share of outcome variance a parameter explains on its own; ST (total-order) also includes the variance it drives through interactions with other parameters. So ST > S1 flags a parameter that matters mainly in combination with others, while ST ≈ S1 means it acts independently. The parameters at the top are where measurement or elicitation effort would most sharpen your conclusions. Wide whiskers mean n is too small — the indices are noisy; raise n (total runs grow as n·(d+2)). As a sanity check, 0 ≤ S1 ≤ ST ≤ 1 up to Monte-Carlo noise, and ΣS1 ≤ 1.
GSA vs the intervention ranking The intervention ranking answers "which lever should I pull?"; GSA answers "which uncertain assumption should I pin down?". A link can be a strong driver of variance (high ST) without being something you can act on — that is a measurement priority, not an intervention.

2Scenario discovery (PRIM)

GSA ranks drivers globally. Scenario discovery instead localises: it draws an ensemble of parameter sets, labels the "cases of concern" (here: peak-hour use stays high despite the full intervention package), and finds an interpretable box of the input space where those cases concentrate.

from sip_systemsinsightpipeline import discover_scenarios
from sip_systemsinsightpipeline.plots import plot_scenario_tradeoff, plot_scenario_box
import numpy as np

X, y = sdm.sample_outcomes(n=1500, seed=2)        # ensemble: inputs X, outcome y
threshold = float(np.quantile(y, 0.8))            # worst 20% = "high peak use"

result = discover_scenarios(X, y, threshold=threshold, direction="above",
                            method="prim", peel_alpha=0.05, min_coverage=0.5)
box = result.box
print(box.limits)        # only the dimensions PRIM had to restrict

On the run shown here PRIM isolates a box with density 0.77 (precision) and coverage 0.51 (recall), restricting only 9 of 94 uncertain parameters (it holds 13% of all draws).

How to read it PRIM reports two numbers for every candidate box. Density (precision) is the share of points inside the box that are cases of concern; coverage (recall) is the share of all concerning cases the box captures. Peeling drives density up by shrinking the box, which inevitably sheds coverage — the trade-off curve lets you choose. Crucially, box.limits lists only the dimensions PRIM had to restrict: those are the parameters whose values jointly produce the bad outcome. Every other input is left unconstrained, and that is the finding — the danger does not depend on them. Read a box as a sentence: "the concern concentrates when these few link strengths fall in these ranges, regardless of everything else."

3CART: the same question as a tree

CART (a classification tree) is an alternative backend. It is good for cross-checking which variables matter and for regions that are not a single box:

res_cart = discover_scenarios(X, y, threshold=threshold, direction="above",
                              method="cart", max_depth=3, min_density=0.7, seed=0)
print(res_cart.splitting_variables)
for rule in res_cart.rules[:3]:
    print(rule)

How to read it The splitting variables are the inputs the tree found informative — they should echo PRIM's restricted dimensions and GSA's top parameters. Each rule is a path to a high-density leaf, reported with the same density/coverage/mass. Where PRIM gives one box, CART can give several rules (useful when the danger lives in more than one corner of the space).

4Putting it together

With the existing intervention ranking and Loops That Matter, these two tools complete SIP's "analyse → find leverage → stress-test robustness" loop: rank the levers, optimize the budget, attribute behaviour to loops, then ask which assumptions the answer hinges on (GSA) and under what conditions it fails (scenario discovery).
