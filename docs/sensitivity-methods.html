Choosing a sensitivity method

SIP offers three global-sensitivity measures. They answer subtly different questions and behave differently near zero. This page helps you pick one, and documents the estimator and confidence-interval theory behind them.

The three families

All three are exposed through SDM.run_GSA(method=…) and operate on the same uncertain parameters (the sampled causal-link strengths and custom-equation parameters), under a chosen intervention scenario.

MethodQuestion it answersWhat it captures Near-zero behaviourCost
Sobol method="sobol"
variance-based		 How much of the outcome's variance does each parameter drive (alone: S1; including interactions: ST)? Effect on the variance only. Decomposes total variance into per-parameter shares. Estimator is a difference of variances → can dip below 0 for an unimportant parameter (handled by projection + BCa, below). n·(d+2) model runs (Saltelli design).
Borgonovo δ method="delta"
moment-independent		 How much does fixing a parameter shift the whole output distribution? Effect on all moments (mean, spread, tails, shape) — not just variance. Sees interaction-only effects that S1 misses. Non-negative by construction. Small positive bias near zero, never negative. n runs (given-data; no special design).
PAWN method="pawn"
CDF-based		 How much does conditioning on a parameter move the output's cumulative distribution? Distributional effect via the Kolmogorov–Smirnov distance between unconditional and conditional output CDFs. Non-negative by construction (a KS distance). n runs (given-data).
Rule of thumb Start with Sobol for the familiar variance decomposition and the S1 vs ST "acts alone vs through interactions" reading. Add Borgonovo δ when you care about effects on the whole distribution (tails, skew) or suspect interaction-driven inputs — δ is non-negative, cheaper, and reuses the same ensemble as scenario discovery. Use PAWN as a robust CDF-based cross-check.

Why a Sobol index can come out negative — and what SIP does

A Sobol index is, by definition, a ratio of variances Si = Var(E[Y|Xi]) / Var(Y) ∈ [0, 1]. Its Monte-Carlo estimator, however, is a difference of variance estimates. For a parameter whose true index is ≈ 0, that difference is a small number plus noise, so at finite sample size it routinely lands slightly below zero. This is a sampling artifact, not a real negative effect, and it shrinks as n grows.

Two facts bound what is possible here:

  • You cannot make the estimator both unbiased and non-negative. Because the index is a ratio, the ratio of two unbiased estimators is itself biased; there is no finite-sample uniformly-minimum-variance unbiased estimator. Lower-variance estimators exist and are asymptotically efficient (Janon et al. 2014; and Owen 2013 specifically targets the noisy small-index regime), which shrinks the negative excursions but does not eliminate them.
  • Imposing the constraint is a modelling choice, not a free lunch. SIP projects the point estimate and its interval onto [0, 1] (the clip=True default), so an uninformative parameter reports S1 = 0 with a one-sided interval like [0, 0.05] ("indistinguishable from zero"). Pass clip=False for the raw estimates — large negative excursions are then a useful, direct signal that n is too small.
If you want to avoid the issue entirely The cleanest answer is to not decompose variance at all. The moment-independent measures (δ, PAWN) are non-negative by construction and measure the shift in the whole distribution — see the tutorial section where Borgonovo δ assigns a clearly positive importance to a parameter whose variance-based S1 ≈ 0.

Confidence intervals: why not a symmetric ± band

Near the 0/1 boundary the sampling distribution of an index is skewed, so a symmetric estimate ± 1.96·SE interval is the wrong shape — it can extend below zero even though the true value cannot. SIP therefore uses an asymmetric bootstrap interval:

IntervalShapeBoundary behaviourUsed by SIP?
Symmetric (Delta-method / normal ±SE)symmetric wrong near 0/1; can cross the boundaryno
Percentile bootstrapasymmetric better, but deformed (biased) near a boundaryci="percentile"
BCa (bias-corrected & accelerated)asymmetric corrects both the bias and the skew at the boundarydefault ci="bca"

The Delta method gives analytic standard errors essentially for free and is fine for large indices, but because the interval it produces is symmetric it does not solve the boundary problem — which is precisely the case we care about. BCa (Efron 1987) is computed from the bootstrap replicates plus a jackknife for the acceleration term; SIP reuses SALib's exact Saltelli / Jansen estimator for the point and replicates.

Advanced / future options

  • Lower-variance point estimators (Owen 2013; Janon et al. 2014) reduce the near-zero noise at the source. Not in SALib's sobol.analyze; a candidate future addition.
  • Gaussian-process (Kriging) emulation (Oakley & O'Hagan 2004) fits a Bayesian emulator to the input–output data and derives the posterior of the Sobol indices. Because every posterior draw is a real function, each draw's true index lies in [0, 1], so the posterior is supported there — physically consistent, with no negativity and no ad-hoc projection. The catch is dimensionality: a GP is O(N³) and degrades badly beyond a few dozen inputs, so for a model like the insulation example (d = 94) it needs screening or dimension reduction first. A rigorous but heavier option, noted here for completeness.
